# 纯前端部署说明

当前版本已经不需要后端：

- 抽卡记录保存在她打开网页的那个浏览器里。
- 抽卡不再按每天 7 点刷新，也没有次数限制。
- 只会抽 `public/static-cards.js` 里的正式内容卡。
- 全部抽完后，页面会显示“已经抽卡结束”。

## 推荐方案：腾讯云 EdgeOne Pages

这是更适合国内访问的 0 成本前端部署方式。

1. 把 `card-draw-system` 上传到一个 Git 仓库。
2. 打开 EdgeOne Pages，选择导入 Git 仓库。
3. 项目设置：
   - 构建命令：留空
   - 输出目录：`public`
   - 环境变量：不需要
   - Edge Functions / KV：不需要
4. 部署完成后，把生成的网址发给她。

## 更新卡牌内容

如果之后继续在 `data/cards.json` 里补卡，运行：

```bash
npm run build:static-cards
```

然后重新部署即可。

## 注意

因为这是纯前端版本，收藏夹和使用记录只存在她当前浏览器的 `localStorage` 里。如果她换手机、换浏览器或清理浏览器数据，记录会重新开始。
